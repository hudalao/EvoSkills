# 研究想法生成（Research Ideation）

从研究目标到排序后的想法，再到一份详细提案。所有论文检索与精读都委托 `paper-navigator`，本技能自身不搜论文。

```
定义目标 → 文献接地(paper-navigator) → 想法树生成(3轴) → 三人格精炼
        → ELO 锦标赛 → 呈现 Top-3 给用户 → 用户选择 → 扩展为提案 → 验证迭代
```

## 何时使用
- 寻找研究方向 / 头脑风暴研究想法
- 评估一个想法是否新颖、是否值得做
- 排序或比较多个研究想法
- 从一个想法生成研究提案

不适用于：检索、搜索、阅读论文 → 一律用 `paper-navigator`。

---

## 第 1 步：定义长期研究目标

从一个兼具科学价值与实用价值的目标出发——足够宏大可支撑多篇论文，又足够具体可指导日常决策。

**用终极形态倒推**：问"这个方向的终极形态是什么？如果问题被彻底解决，世界会变成什么样？"再从那个愿景倒推出沿途必须解决的里程碑任务——这些中间任务就是真正重要的问题。

**目标驱动优先**：挑一个重要任务死磕，方法不行就换、太难就略放宽约束——总能产出有用结果；避免"在基准 Y 上刷 0.5%"的想法驱动陷阱。

**承诺前自检（问题选择清单）**：□ 连着重要长期目标 □ 能具体说出现有方法在哪/为何失败 □ 成熟方案检查过关（Level 3/4，见反直觉规则 3）□ 有数据或能造数据 □ 资源与专长内可行 □ 至少有一个可行解法路径。

## 第 2 步：文献接地（经由 paper-navigator）

**调用 `paper-navigator`** 收集 30-50 篇相关论文。不要跳过此步，也不要用通用知识替代——想法必须扎根于真实论文。

### 四种新颖度类型
每篇论文的贡献都落在四类之一，理解这个层级能看清自己的工作该处在哪。
- **Type 1**：定义里程碑任务的开创性工作（新颖度最高，影响最持久）
- **Type 2**：提出全新流水线或表示范式
- **Type 3**：引入新模块（注意力机制、损失函数、训练策略）
- **Type 4**：用已有模块/技巧做增量改进（最易产出，最难令人记住）

**用新颖度层级找空白**：缺失的里程碑任务（领域还没形式化的重要问题，Type 1 机会）；Type-2 稀少的任务（范式单一、待开拓）；可把某任务成熟的 Type-2 范式迁移到另一任务。最高产的想法常出现在"新颖度空白 × challenge-insight 空白"的交点。

### 构建 challenge-insight 树
对收集的论文，建立**技术挑战 ↔ 应对它的洞见/技术**的多对多映射：
- **抽取挑战**：每篇论文解决什么技术问题？（常在引言第 2-3 段、相关工作、方法动机里）
- **抽取洞见**：它用了什么关键技术或思路？
- **建立连接**：哪些洞见应对哪些挑战？

**它如何驱动想法生成**：
- 挑战对应的洞见很少 → **未解问题**（第 3 步的候选）
- 某洞见尚未被用到某挑战上 → **跨域迁移机会**（第 4 步的候选）
- 挑战对应的洞见很多 → 已被充分研究，除非有根本性新角度否则避开

*示例*：挑战「长上下文显存爆炸」← KV 缓存量化／滑窗稀疏注意力／状态空间模型；挑战「长程依赖丢失」← 稀疏注意力／状态空间模型。可见"稀疏注意力"一技多用；而某洞见尚未被试于某挑战，即是迁移机会。

**执行铁律**：不要在没有真实论文接地时生成想法。树中必须引用真实论文（标题、作者、发现）。

### 论文阅读深度
精读交给 paper-navigator，按目的选深度：①**技术级**（要复现或在其上构建）——读全方法与公式；②**分析级**（多数情况）——读引言第 2-3 段与方法概览，能用 2-3 句讲清它解决什么、为何更好；③**语境级**——读摘要/引言/结论，定位其新颖度类型与在文献树中的位置、找出一处局限或失败场景。多数论文只需 ②③，① 留给要直接构建的。

**每篇记 5 项摘要**（喂给两棵树）：①一句话新颖点；②流水线（步骤1做X/2做Y/3做Z）；③问题与动机（解决什么/现有为何失败/为何更好）；④文献树位置（新颖度类型/建立在什么上/使能什么）；⑤局限与失败场景（什么数据或假设会让它崩）。

## 第 3 步：用想法树生成候选（3 轴展开）

以目标为种子根节点，**每层只变化一个轴**，避免"相似但不同"的伪多样性。目标产出 **15-21 个叶子候选**（<10 个多样性不足，锦标赛不可靠）。

- **第 1 层·技术轴**：解决问题的核心技术路径。每个分支必须是**根本不同的范式**（依赖不同原理，而非不同参数）。问"有哪 3 种根本不同的方式解决这个问题？"取 3 个。
- **第 2 层·领域轴**：技术应用的语境/数据域。每个领域引入**实质不同的约束**。问"在哪些语境下这个技术会面临根本不同的挑战？"每节点取 2-3 个。
- **第 3 层·形式轴**：精确的问题设定——输入、输出、约束、评测标准。每节点取 1-2 个。

**示例（种子：跨血脑屏障给药）**：技术轴=纳米颗粒载体／受体介导转胞吞／聚焦超声；领域轴（以纳米颗粒为例）=阿尔茨海默(慢性低剂量)／胶质母细胞瘤(急性高剂量)／儿童癫痫(发育期脑)；形式轴（以胶质瘤为例）=最大化肿瘤富集／最小化全身毒性／影像-治疗一体。每条根→叶路径即一个明确、独立的研究问题。

**逐节点循环：提出 → 评审 → 精炼**——写 2-3 句描述（做什么/为何可能成立/与父节点的区别）；评审三问（是否真不同于兄弟节点 / 是否至少可行 / 描述是否够具体可执行）；据评审去掉模糊措辞、把新颖性说具体。

**剪枝准则**：仅当①明显不可行（需根本拿不到的资源、违反已确立的理论/物理约束）或②与其它分支重复时才剪。**不要剪**"看似不太可能但未被证伪"、超出你当前专长、或挑战常识的想法。

**进锦标赛前核对多样性**：①层间——第 1 层各分支是否真是不同技术（能否一句话向同行说清、且被认作不同路径）；②层内——每个技术下的领域是否带来不同技术挑战；③叶子——不同叶子的评测指标是否会不同。任一不达标则补充或替换分支。

**两种贡献形态**（每个想法选其一）：**聚焦型**——单个新组件 + 干净可检验假设，易验证、归因清晰；**系统型**——组件间紧密因果交互、产生涌现能力，潜力大但须警惕变量混淆。

### 想法候选格式
```
# 研究想法：[精炼标题]
## 核心想法
[一段话：提案内容 + 对应的技术-领域-形式]
## 新颖性论证
[2-3 句：与哪篇/哪类已有工作不同，差异具体在哪——点名所区别的先前工作]
## 可行性
[所需资源/数据、粗略周期、已知风险]
## 验证计划
[具体实验提纲：数据集、基线、指标]
```
（进锦标赛后追加记录：初始 ELO 1500、当前 ELO、胜/平/负。）

## 第 4 步：精炼想法（三人格 × N 轮）

对每条初始想法各开一条精炼轨道**并行**推进；每条轨道跑 N≈3 轮：①评估当前最佳（新颖/可行/影响/对齐）→ ②**三个人格**各据评估出一版精炼 → ③取最佳作下一轮种子。轨道冠军 = 该轨道历轮最佳；所有轨道冠军进第 5 步锦标赛。

| 人格 | 关注 |
|------|------|
| **创新者** | 新颖与创造力——突破性、高风险高回报 |
| **实用者** | 可行性——现实、可清晰执行 |
| **批判者** | 科学价值——推进理解、严谨 |

### 5 种进化策略（括号内为触发条件）
1. **接地强化**（机制缺文献支撑时）：用文献引用夯实
2. **改善内聚**（机制有逻辑漏洞时）：修复
3. **启发与组合**（想法单薄时）：与文献中另一概念组合
4. **简化**（变量过多/假设不清时）：剥离为干净、可检验的假设
5. **文献驱动转向**（评估判定死胡同时）：放弃当前机制，从文献提出新路径

**pivot 硬规则**：若评估判定该路径是死胡同，人格**必须转向**——精炼不局限于打补丁。

### 逻辑内聚原则
- **变量过多** → 用**减法聚焦**：隔离最有希望的那个变量
- **组件脱节** → 用**强相关**：建立显式因果链

### 设计方案的两种模式
设计前先从四个来源蒸馏知识（按可得性排序）：①**论文**——搜解决相似问题的工作，重点读引言第 2-3 段对问题成因的分析与方法的关键洞见（同一难题在不同领域常用不同术语）；②**综合**——用自己的话重述、形成"为何难"的判断，区分哪些已有洞见适用于你的设定（原创思考在此发生）；③**他人**——与资深研究者讨论你的分析（讲出来本身就在深化）；④**实验**——用小实验验证对失败成因的假设，再设计完整方案。

价值在于理解技术"为何"奏效再创造性组合，而非简单拼接——"让工具奏效"难，"套用工具"易。
- **模式A·跨域迁移**（对应成熟方案检查 Level 3）：剥离领域细节、抽出技术内核 → 广搜其它领域中技术上相似的问题 → 理解其解法**为何**成立 → 适配本领域（数据/模态需改什么、哪些组件可直接迁移、哪些需重设计）→ 在代表性样例上先验证。源领域越远，迁移越显新颖，但适配工作也越多。
- **模式B·问题分解**（对应 Level 4）：拆成独立子问题并理清依赖 → 每个子问题套用模式A → 仍无解则继续分解或放宽约束 → 逐组件增量组合并测试接口兼容。
- **创造性组合**：组合前**枚举所有可能的方法与流水线顺序**，逐一分析权衡（精度/效率/假设/失败模式），据分析而非直觉择优，并对至少一个备选做消融。投入实现前先查这个组合是否被试过——若试过，你的洞见必须针对它失败的原因。
- **设计自检**：□ 能解释方案"为何"成立（不只是"做什么"）□ 针对一个具体技术瓶颈（非笼统改进）□ 查过这个组合没被做过 □ 关键假设已用小实验验证 □ 能说出相对最明显备选的优势 □ 资源内可实现。若拆不出子问题或找不到子解，多半是认知差距（多读远域论文）或检索差距（换术语再搜），真正世界级难题罕见。

## 第 5 步：ELO 锦标赛 → 呈现 Top-3

对所有候选两两比较排序，再把 **Top-3 呈现给用户选择**。

**四个维度**（各 1-10 分）：
- **新颖性**：与已发表工作有多不同？（9-10 全新路径 / 7-8 非显然组合 / 5-6 已知技术用于新设定 / 3-4 增量 / 1-2 复制）
- **可行性**：现有资源内能否实现验证？（9-10 2-3 月内可完成 / 5-6 需大量投入 / 1-2 资源不现实）
- **相关性**：是否解决与目标对齐的重要问题？（9-10 领域 top-3 开放问题 / 5-6 相关但非最紧迫 / 1-2 基本无关）
- **清晰度**：是否定义到可立即上手？（9-10 问题与指标都明确 / 5-6 方向清楚细节待查 / 1-2 不清楚问题是什么）

**ELO 机制**：起始分 1500，K=32。每场两两比较 → 期望分 `E_A = 1/(1+10^((R_B−R_A)/400))` → 更新 `R_A' = R_A + K×(S_A−E_A)`（胜 S=1，平 0.5，负 0）→ 按终分排序。两两比较靠推理给四维打分，无需脚本。候选 15-21 个时约跑 4-5 轮配对（每轮按当前分就近配对、避开已交手过的）；若某想法 2-3 轮即冲到 1700+，多半是真强，但要核对它是否战胜了强对手。**收敛检查**：每轮后看 Top-3 是否与上轮相同——相同即可提前停；Top-5 仍大幅变动则再加一轮；满 5 轮无论如何停。

**每场打分卡**：A/B 各维度 1-10 + 一句理由 → 各自复合分 → 定胜者 → 按上式更新双方 ELO。*算例*：A、B 均 1500，A 复合 7.75 > B 7.25：E_A=1/(1+10⁰)=0.5，A 胜 S=1 → R_A'=1500+32×(1−0.5)=**1516**，R_B'=**1484**。

**呈现 Top-3**：① 对比表（排名/标题/核心机制/四维分/ELO），示例：

| 排名 | 标题 | 核心机制 | 新颖 | 可行 | 相关 | 清晰 | ELO |
|------|------|----------|------|------|------|------|-----|
| 1 | 分层 KV 量化 | 按层敏感度分配位宽 | 8 | 8 | 9 | 8 | 1276 |
| 2 | 滑窗+检索增强 | 局部稠密+远程稀疏召回 | 7 | 9 | 8 | 7 | 1242 |
| 3 | 状态空间混合 | SSM 替换深层注意力 | 9 | 6 | 8 | 7 | 1210 |
② **每个 Top-3 的完整精炼版**（用第 3 步格式 + 一段"精炼小结"：简化/新增了什么、哪个人格驱动了关键改动、补了哪些风险缓解）——此部分必须给，不可只给对比表；③ **跨方向分析**：三者的共同主线（可能提示组合方案）、共享的维度强弱模式、以及原种子里三者都没覆盖的方面；④ 选择提示："想把哪个想法扩展成完整提案？（1/2/3，或组合要素）"

## 第 6 步：扩展为研究提案

用户选定后，扩展为一份手稿级提案。**两阶段**：

**阶段一·先生成领域专属模板**：从研究目标与文献识别领域；以通用章节（摘要/问题/相关工作/方法/评测/结论）为基础，叠加领域专属章节并适配术语（如医学用"研究设计"、社科用"方法论"）。

| 领域 | 通用章节外追加 |
|------|----------------|
| 医学/临床 | 伦理与 IRB、试验设计(分期/分组/终点)、入排标准、统计功效与样本量、安全监测 |
| 社科 | 理论框架、招募策略、量表/访谈设计、编码方案、研究者立场 |
| 化学/材料 | 合成路线与反应条件、表征计划(XRD/SEM/NMR)、安全危害分析、放大考量 |
| 物理 | 数学推导/理论框架、计算设置(DFT 参数/网格)、误差与不确定度分析 |
| 工程 | 设计规格与约束、仿真/FEA、原型制造、测试标准合规、成本效益 |
| 生物 | 模型系统论证、IACUC 伦理、对照设计、生物学重复与统计功效 |
| CS/ML | 算力需求、基线复现计划、消融设计、代码/数据可得性声明 |

**阶段二·填写提案**。六段结构：①**背景**——精确问题定义(输入/输出/约束/评测)+现有方法在何处为何不足+范围边界；②**相关工作**——定位+3-5 篇最相关+差异化(叙事而非罗列)；③**方法**——核心洞见+步骤+关键区分点+假设；④**实验计划**——数据集(为何选)/基线(≥3 含 SOTA)/主次指标/消融设计(对应贡献 2)；⑤**预期结果**——定量目标(给区间、可证伪)+定性预测；⑥**风险与缓解**——技术/资源/评测三类风险，各配后备方案。

写作遵循：
- 面向领域顶级审稿人写——每个论断有支撑、每个设计有理由、主动回应弱点
- **核心洞见陈述法**：明写"我们的核心洞见是 X，它使 Y 成为可能，因为 Z"。洞见不是方法的复述——"剪枝让模型更小"是定义；"不同层的注意力头对不同输入长度敏感，使逐层差异化剪枝率成为可能"才是洞见。
- **三贡献法则**：恰列 3 条贡献（1 条显新颖不足，2 条不完整，4+ 条不聚焦），且每条**可检验**（能设计实验判定其成立）
- 避免变量混淆：清楚指出哪个是核心贡献，回答"如何确保观测效应来自你声称的创新而非混杂因素"
- **问题陈述避三坑**：太宽（"提升 LLM 效率"→"单卡 A100 上把 7B 推理延迟降 ≥40% 且 MMLU 保持 ≥95%"）；无基线参照（"现有方法慢"→"SOTA 的 SpecDec 达 2.3× 加速但需草稿模型"）；缺约束（"更快推理"→"在无辅助模型、≤16GB 显存下"）。
- **各段写作要诀**：摘要——单段密集，按"领域+挑战→现有局限→你的方法→验证→预期影响"流；相关工作——叙事呈现领域演进，构造让你的工作显得"必然下一步"的 gap 论证，附范式对比表（非罗列论文）；方法——每个组件按"目的→机制→为何这样设计克服了某具体局限"写，配一个示例；评测——按研究问题组织，每问对应实验设计+预期结果+备选结局。
- **形式化适度**：定量领域用数学记号、实验领域用精确协议描述——但形式表达后必跟一句直觉解释。
- **引用格式**：`**1. 标题** (年). _作者_. *会议/期刊*. 被引: N. [链接]`；所有技术论断都标来源 [1][2]。

## 第 7 步：验证与迭代

在代表性数据上跑实验、验证关键假设。**失败要分类**：实现性失败（接口/超参/数据问题）→ 修复重试；根本性失败（机制不成立）→ 带着新认知返回第 4 步转向，或返回第 1 步换问题。新观察到的失败案例本身可能就是一个新问题（见反直觉规则 2）。

---

## 反直觉规则
1. **选问题 > 设计方案**：选"做什么"比"怎么做"更重要
2. **追新失败案例，而非增量改进**：把现有方法用到新数据/更难输入上，找它崩溃的设定——**新失败案例本身就是可发表的贡献**（如 NeRF 在静态场景有效，用到形变的手术视频就失败，这定义了一个新问题）
3. **若已有成熟方案，就换问题**：成熟方案检查四级——①同任务已有好解→换；②跨域相似技术核已被解→换；③仅 1-2 个远域有解→可做（迁移机会，适合新手）；④无任何领域有解→可做（高风险高回报，适合专家）
4. **技术是创造性组合，不是拼接**：简单 A→B 流水线不算贡献；组合前**枚举所有可能再据分析择优**
5. **生成阶段先量后质**：先大量产候选再评估
6. **可行性不是可选项**：精彩但不可行的想法浪费研究周期
7. **相信锦标赛胜过直觉**：排序常带来惊喜
8. **新锤子机会**：出现突破性工具时，把它用到**你路线图的里程碑任务**上——而非在它原基准上改进它（那是在与工具创造者竞争、且空间拥挤）。公式：突破工具 + 你的里程碑任务 = 高影响论文
